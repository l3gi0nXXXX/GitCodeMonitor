# GCM GitCode Maintainer Mention Format v1

`contractVersion`: `gcm-metis-gitcode-review-v1`

GCM owns maintainer mention formatting during GitCode writeback. Metis generated replies must not include team leader or codeowner mention footers, bot attribution, hidden markers, `auto-reply`, or `gitcodemonitor` branding text.

## Rules

- Use only valid resolved mentions such as `@alice`.
- Mention source order is repo team leaders first, then PR CODEOWNERS.
- Merge team leaders and codeowners, then deduplicate case-insensitively.
- Join mentions with the Chinese comma `，`.
- Append directly after the draft with one newline and no blank line.
- If no team leader or codeowner mention is resolved, append nothing.
- If the same natural sentence already exists, append nothing.
- Do not expose diagnostics, raw JSON, tokens, or local paths in the footer.

## Sentence Pool

- `麻烦 {mentions} 一起看看。`
- `{mentions} 看看，以你们确认的结果为准。`
- `这块可能还需要 {mentions} 帮忙补充一下背景。`
- `麻烦 {mentions} 看看这个现象是否和你们那边的判断一致。`
- `也想听听 {mentions} 对这个方向的看法。`
- `{mentions} 帮忙看看这块代码，我这边先给一个初步判断。`
- `这个补充也麻烦 {mentions} 一起看看。`

GCM chooses a sentence deterministically from event id, mention list, and template pool version so retries remain stable.

## Forbidden User-Visible Text

The final GitCode comment must not include:

- `auto-reply`, `auto reply`, or `automatic reply`
- `bot`, `generated by`, `AI generated`, `GitCodeMonitor`, or `gcm`
- hidden HTML markers such as `<!-- ... -->`
- `---`
- role labels such as `@Team Leader:` or `@Codeowner:`
